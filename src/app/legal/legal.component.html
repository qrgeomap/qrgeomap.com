About the data:

This application allows to open and use maps, showing the user's live location on the loaded map. The user's location is not saved or sent anywhere at any time; It is simply shown to the user on the map while using it.

The application also allows to generate maps for download and/or publication on this website.

To generate a map it is necessary to provide a track (GPX file). The user must be the owner of that GPX file or have sufficient rights for its use and publication, being exclusively responsible for it.

The generated map will include the track uploaded by the user drawn and (optionally) the waypoints included. Additionally, a title (optional and editable by the user) may be displayed at the bottom of the generated map.

The resulting map will contain an overlaid QR code, which includes a link to a URL, to which the georeferencing (coordinates) of the map is added (in its parameters).

The user can enter the URL they want for the QR link, unless they choose the option to "Publish" the map (in which case the URL will be from the domain qrgeomap.com).

If the user decides to publish a map, they should know that all the information they enter will be completely public.

Together with the image of the map that is published, QRgeomap.com will save the following information in its database:

  • Title of the map.
  • Map coordinates.
  • Creation date.
  • Unique identification key*.

The owner of QRgeomap.com may make any use he deems appropriate of this data.

At no time is any personal information required.

* When the user publishes a map, they obtain a URL for public use and at that moment they are also provided with a key or unique identifier for the map. The user must save this key if they want to be able to exercise any type of future rights over the generated map (e.g. request its deletion).

Any request will be made by email, addressed to: mail@lorenzomoreno.com

Sobre los datos:

Esta aplicación permite abrir y usar mapas, mostrando la ubicación en tiempo real del usuario sobre el mapa cargado. La ubicación del usuario no se guarda ni envía a ningún sitio en ningún momento; simplemente se muestra al propio usuario sobre el mapa mientras lo usa.

La aplicación también permite generar mapas para su descarga y/o publicación en esta web.

Para generar un mapa es necesario aportar un track (archivo GPX). El usuario deberá ser propietario de ese archivo GPX o tener derechos suficientes para su uso y publicación, siendo responsable exclusivo de ello.

El mapa generado incluirá dibujado el track cargado por el usuario y (opcionalmente) los waypoints que incluyese. Además podrá mostrarse al pie del mapa generado un título (opcional y editable por el usuario).

El mapa resultante contendrá un código QR superpuesto, que incluye un enlace a una URL, a la que se añade (en sus parámetros) la georreferenciación (coordenadas) del mapa.

El usuario podrá introducir la URL que desee para el enlace del QR, salvo si elige la opción de "Publicar" el mapa (en ese caso la URL será del dominio qrgeomap.com).

Si el usuario decide publicar un mapa debe saber que toda la información que introduzca será totalmente pública.

Junto a la imagen del mapa que se publica QRgeomap.com guardará en su base de datos la siguiente información:

  • Título del mapa.
  • Coordenadas del mapa.
  • Fecha de creación.
  • Clave de identificación única*.

El propietario de QRgeomap.com podrá hacer cualquier uso que considere oportuno de estos datos.

En ningún momento se requiere ningún dato personal.

* Cuando el usuario publica un mapa obtiene una URL para su uso público y además en ese momento se le facilita una clave o identificador único del mapa. El usuario deberá guardar esta clave si quiere poder ejercer algún tipo de derecho futuro sobre el mapa generado (ej: solicitar su eliminación).

Cualquier solicitud se hará por email, dirigiéndose a: mail@lorenzomoreno.com
